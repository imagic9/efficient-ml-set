# Model card — M0, FP32 baseline

**One sentence:** MobileNetV2 classifying camera-trap frames into 16 classes, whose
bobcat score drives a photographer's shutter trigger on a Raspberry Pi 5; this FP32
model is the baseline every optimized candidate (M1-M4) is measured against.

| | |
|---|---|
| Architecture | MobileNetV2, width 1.0, ImageNet-pretrained (torchvision), classifier replaced with a 16-output linear layer |
| Input | `[1, 3, 192, 256]` NCHW float32 (256x192 WxH, frozen by C1a) |
| Output | 16 logits; softmax-normalised, `bobcat` score compared to the calibrated threshold |
| Deployable artifact | `m0_fp32_seed42.onnx`, ONNX opset 17, FP32 |
| Training run | `c2_m0_fp32_seed42_20260716T061203Z`, best epoch 11 of 17, 16m09s on gx10 |
| Policy | `artifacts/policies/bobcat_v1.json`, threshold **0.538088**, status **`recall_floor_infeasible`** |

## Intended use

A field assist for a wildlife photographer: the device watches the camera's view and
closes a shutter-release circuit when a bobcat is likely in frame, so the photographer
does not sit motionless for hours. The 5% per-domain false-fire budget (DESIGN §6.3)
is a product constraint: every false trigger spends a frame, an actuation, and battery.

**Out of scope.** Population monitoring, ecological research conclusions, enforcement,
or any use where a missed or false detection carries consequences beyond a wasted
frame. Scores are not calibrated probabilities (DESIGN §6.4) and the trans-val
evidence below is from a single unseen camera location.

## Data

- **CCT-20** (Caltech Camera Traps, 20-location benchmark subset), via
  [lila.science](https://lila.science/datasets/caltech-camera-traps), using the
  cleaned development split of DESIGN §5.1.1: the official cis/trans validation and
  test sequence overlaps are removed from the development sets; test sets stay sealed
  (§5.4) and were not touched by training, calibration, or this card.
- **`cct_empty_train_v1`**: 5,000 empty frames from CCT locations disjoint from all
  20 CCT-20 locations (§5.2), training only. The supplement's inclusion won the C1a
  ablation on real validation data.
- Splits used by this model: `train` (+ supplement), `cis-val-clean`, `trans-val`.
  Dataset manifests and their hashes are recorded in the run's `hashes.json` and
  inside the policy's `calibration.datasets`.

## Preprocessing (the frozen contract, DESIGN §5.5)

1. Decode JPEG as 8-bit BGR (OpenCV); 2. BGR→RGB; 3. aspect-preserving resize into
256x192; 4. center-pad with RGB (114,114,114); 5. float32, /255; 6. ImageNet
normalisation; 7. HWC→NCHW.

P1 parity: the C++ fused implementation is **bit-exact** against Python on all 25
golden fixtures (OpenCV 4.13 vs 4.6); the reference C++ path is within 7.15e-07.

## Metrics

All numbers are validation; test is sealed until F-phase. Frame F2, recall,
precision are at the named threshold; event capture = positive sequences with ≥1
trigger / all positive sequences.

|---|---:|---:|
| bobcat F2 | 0.6272 | 0.1054 |
| frame recall | 0.7361 | 0.0870 |
| precision | 0.3941 | 0.6900 |

### At the calibrated operating point 0.538088 (`bobcat_v1`, DESIGN §6.3)

| | cis-val-clean | trans-val |
|---|---:|---:|
| bobcat F2 | 0.6355 | 0.0965 |
| frame recall | 0.7361 | 0.0794 |
| precision | 0.4109 | 0.6923 |
| sequence-balanced recall | 0.7333 | 0.0792 |
| false-fire | 0.0495 | 0.0300 |
| event capture | 0.8600 | 0.1660 |

**The primary rule is NOT met.** No admissible threshold (≤5% false-fire on both
domains) reaches 90% sequence-balanced recall on both; the registered status is
`recall_floor_infeasible`, and no table or summary may present the shipped operating
point as a pass. The seq_id-cluster bootstrap of the calibration rule (1,000
replicates) puts the threshold in **[0.415, 0.704]**; every replicate returned the
same `recall_floor_infeasible` verdict.

### Training variability (C5 confirmation seeds, frozen recipe)

Seeds 17 and 73 re-ran the identical §7.2 recipe (verified by
`validate.seed_variability`); seed 42 remains M0 by pre-registration.

| F2@0.5 at the selected checkpoint | seed 17 | seed 42 (M0) | seed 73 | mean ± std (n=3) |
|---|---:|---:|---:|---:|
| cis-val-clean | 0.6130 | 0.6272 | 0.6716 | 0.6373 ± 0.0305 |
| trans-val | 0.1343 | 0.1054 | 0.1029 | 0.1142 ± 0.0175 |

No phase-B epoch of any seed exceeded trans F2 0.135 (40 epochs across three
seeds), which answered issue #18: M0's weak unseen-camera recall is a property of
the frozen recipe, not bad luck with seed 42. Full report:
`results/training/c5/seed_variability.md`.

## Limitations

- **Unseen-camera recall is poor and honestly reported** (registered outcome,
  DESIGN §18): at the operating point the device catches ~86% of bobcat visits at
  seen cameras and ~17% at the unseen trans-val camera. Trans-val is a single
  location — evidence for that location, not a universal estimate.
- **Scores are not calibrated probabilities**; the threshold is meaningful only with
  this model, this preprocessing, and this class map (the policy binds all three by
  hash).
- **Scoring regime** (issue #30): the committed `predictions.npz` for this run holds
  cuDNN-TF32 batch-64 scores (up to 7.25e-3 from deployed FP32 near the threshold).
  Since the DESIGN §6.3 amendment (2026-07-16), score dumps run in the deployment
  regime (TF32 off) and record it in the file.
- Daytime RGB and night IR frames are both in CCT; no separate IR evaluation
  stratum exists yet beyond the P1 synthetic supplement.
